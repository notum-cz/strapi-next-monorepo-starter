# Documentation Overhaul

## What This Is

A comprehensive documentation rewrite for the Strapi + Next.js monorepo starter. The existing Docusaurus site has 8 docs covering core topics (auth, API client, page builder, types, schemas, hierarchy, commands, architecture overview), but significant parts of the system are undocumented. The goal is to map the entire repository, restructure the Docusaurus site for discoverability, and fill all documentation gaps — producing both onboarding guides and deep reference material for new and existing contributors.

## Core Value

Every part of the monorepo has clear, navigable documentation with real code examples so that a developer can understand any subsystem without reading source code.

### Out of Scope

- API reference for every exported function — conceptual + patterns depth, not exhaustive API docs
- Video tutorials or interactive walkthroughs — text and code examples only
- Modifying application code — documentation only, no refactoring
- Fixing known bugs surfaced in codebase mapping — document them, don't fix them

## Context

- Docusaurus 3.9.0 is already initialized at `apps/docs/` with local search, versioning (current version 3.1.1), and GitHub Pages deployment
- Sidebar uses `autogenerated` mode — file/folder structure determines navigation
- Existing docs have no `sidebar_position` frontmatter — ordering is alphabetical by default
- Existing docs are generally well-written but flat (no folder hierarchy, no sections)
- READMEs exist at root, `apps/strapi`, `apps/ui`, `packages/design-system`, `packages/strapi-types`, and `qa/tests` — these should remain as lightweight intros
- Real code examples from the repo are preferred over simplified snippets
- The codebase map at `.planning/codebase/` provides comprehensive analysis of all subsystems

## Constraints

- **Output location**: All docs written to `apps/docs/docs/` — must work with Docusaurus autogenerated sidebar
- **No code changes**: Documentation only — don't modify application source code
- **Existing content**: Migrate and improve existing 8 docs, don't discard them
- **Cross-linking**: Docs should link to each other where topics intersect
- **Frontmatter**: Every doc needs `sidebar_position` and `sidebar_label` for proper ordering
